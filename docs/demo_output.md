# 実行デモ — AIエージェント判定の生ログ

`scripts/demo_judge.py` を実 API キーで実行した結果。スコア3点の案件3件をAIエージェントに判定させた。

実行コマンド:
```
ANTHROPIC_API_KEY=sk-ant-... python scripts/demo_judge.py --limit 3
```

実行モデル: `claude-haiku-4-5` (tool use 有効)

---

## ケース1: 移住・定住支援窓口業務 (中土佐町)

```
🔔 新着案件: 3点

📋 中土佐町移住・定住総合支援窓口業務委託公募型プロポーザルの実施について
🏛️ 中土佐町（高知県）
📅 公告日: 2026-02-16
🏷️ 種別: プロポーザル
🔑 キーワード: 業務委託, プロポーザル(種別)
🔗 https://www.town.nakatosa.lg.jp/life/detail.php?hdnKey=1915

🤖 AI判定: ❌ スキップ推奨（信頼度 72%）
   理由: 移住・定住窓口業務は当社の得意領域（DX/BI/分析）との関連が薄く、
        詳細確認不可。窓口業務委託は通常継続的スタッフ配置を要求され、
        1-2名体制では対応困難の可能性が高い。
   懸念: 窓口業務が主体であれば当社の得意領域外
       / 継続的な人員配置が必要な可能性が高い
       / 詳細ページ表示期間終了で要件確認不可
       / 当社体制では実施難度が高い可能性

[ツール呼び出し回数: 2回]
```

→ ルールフィルタ単体だと「業務委託 + プロポーザル」で3点に乗ってしまうが、エージェントは「**窓口業務 = 継続スタッフ配置**」が当社の体制(1-2名)と合わないことを言語化して却下している。

```
🔔 新着案件: 3点

📋 【公募型プロポーザル実施】仁淀川町まちづくり総合計画（仮称）策定支援業務委託
🏛️ 仁淀川町（高知県）
📅 公告日: 不明
🏷️ 種別: プロポーザル
🔑 キーワード: 業務委託, プロポーザル(種別)
🔗 https://www.town.niyodogawa.lg.jp/life/life_dtl.php?hdnKey=3147

🤖 AI判定: ❌ スキップ推奨（信頼度 75%）
   理由: まちづくり総合計画策定支援は当社の戦略企画コンサル領域外。
        工数規模も大型の可能性。詳細ページ非公開で要件確認困難。
   想定工数: 6〜12ヶ月（推定）
```

```
📋 第3次まんのう町健康増進計画及び食育推進計画策定支援業務委託に係る公募型プロポーザルの実施について

   理由: 健康増進・食育計画の策定支援は政策企画コンサル領域。
```

## 何が嬉しいのか

- **ルールフィルタだけでは取りこぼせなかった**: 3件すべて「業務委託 + プロポーザル」で3点に乗っていた = 通知対象になっていた。AI判定が無ければこれら全部Slackに飛んできて、人間がノイズ処理する手間が発生していた
- **理由が言語化されている**: 単に「不適合」ではなく「窓口業務だから / 政策企画だから / 医療分野だから」と具体的に説明できている → 後で振り返れる
- **必要なところでだけツールを呼ぶ**: ケース2,3は1回 (詳細ページ確認)、ケース1は2回。「常に呼ぶ」ではなく「判断材料が足りない時だけ」という挙動になっている
